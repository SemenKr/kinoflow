Ты выполняешь безопасный рефакторинг кода в проекте React + TypeScript + Vite.

## Когда использовать

Используй этот skill для:
- уменьшения дублирования
- упрощения логики
- улучшения читаемости
- выноса небольших helper / hook / util

Не используй для:
- новых фич
- изменения UX
- изменения API
- архитектурных перестроек

---

## Правила

- Отвечай на русском языке.
- Не меняй поведение кода.
- Не меняй API, UX, тексты и flow.
- Не добавляй новые фичи.
- Делай только минимальные изменения.
- Не трогай несвязанный код.
- Если рефакторинг не даёт заметного упрощения, прямо скажи, что изменения не нужны.
- Всегда проверяй, нет ли уже существующего helper / hook / util перед созданием нового.
- Не создавай абстракции “на вырост”.

---

## Критические проверки перед изменением

1. Не меняется ли поведение компонента или функции.
2. Не меняются ли:
    - side effects (useEffect, запросы, навигация)
    - порядок вызовов
    - условия рендера
3. Сохраняются ли UI-состояния:
    - loading
    - error
    - empty
4. Не ломаются ли зависимости между состояниями.

Если есть риск — явно укажи его до изменений.

---

## Предпочитаемые изменения

- убрать дублирование условий
- упростить ветвления
- вынести повторяющийся стиль
- улучшить naming
- сократить шум в JSX
- сделать код более линейным и читаемым

---

## Чего избегать

- новых слоёв абстракции
- перемещения кода между модулями без причины
- изменения публичных интерфейсов
- смешивания рефакторинга с новой логикой
